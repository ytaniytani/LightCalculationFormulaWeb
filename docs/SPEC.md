# ライティング計算 インタラクティブ学習サイト 仕様書

**プロジェクト名(仮):** Light Calculation Formula Web
**バージョン:** 0.3(間接光=スカイライト/SHライトプローブを追加)
**最終更新:** 2026-07-12

## 1. プロジェクトの目的

ゲームのリアルタイムCGで使われるライティング計算(Lambert、スペキュラー、PBRのBRDF、さらに間接光=スカイライト・SHライトプローブまで)を、**中学数学の知識だけを前提に、数学が得意でない人でも直感的に理解できる**インタラクティブなWeb学習ページを作る。

直接光(ライトから面へ直接届く光)だけでなく、**間接光(壁や床で跳ね返って回り込む光)** までを扱うことで、「なぜ直接光だけの絵はのっぺりするのか」「実際のゲームがどう安く豊かな陰影を出しているか」までを一本道で理解できるようにする。ただしレイトレーシングのような重い厳密計算は扱わず、リアルタイム向けの近似(スカイライト、事前計算したSHライトプローブ)に絞る。

### 1.1 解決したい課題

- ライティングの解説記事は数式が先に出てきて、文系・数学非専門の学習者が挫折しやすい
- 数式の「意味」(パラメータを変えると見た目がどう変わるか)が、静的な記事では体感できない
- 三角関数・ベクトル・内積などの前提知識(高校数学範囲)が「知っている前提」で省略されがちで、中学数学までの学習者が積み上げられる教材がほぼ無い
- 間接光(スカイライト・ライトプローブ・SH)は解説がとくに数式偏重で、「結局それが何をしているのか」の直感を得られる入門教材がほとんど無い

### 1.2 このサイトのアプローチ

1. **中学数学からスタート** — 前提知識は中学数学のみ。しかも「忘れかけ」を想定して、必要な部分は復習から入る。高校数学の内容(三角関数・ベクトル)は、このサイト内で必要な部分だけをゼロから教える
2. **触って理解する** — すべての数式にスライダー付きのデモを併設し、「動かす→変化を見る→式の意味がわかる」の順で学ぶ
3. **一段ずつの階段** — 各節で新しく増える概念は原則1つだけ。いきなり2つ以上の新記号が出てくるページを作らない
4. **数式は「読み方」から** — 記号の読み方、各項が何を表すかを日本語で逐語的に説明する(数式アレルギーを起こさせない)
5. **見た目と式を常に対応させる** — 3Dプレビュー・2Dグラフ・数式の3つを同じ画面に置き、どれかを操作すると他も連動する

## 3. 学習設計の原則(易しさを担保するルール)

カリキュラムの並びと書き方は、以下のルールで統一する。

1. **難易度の階段を明示する** — 全節に難易度バッジを付ける
   - ★☆☆ = 中学数学の復習
   - ★★☆ = このサイトで初めて学ぶ数学(高校範囲を噛み砕いたもの)
   - ★★★ = CG特有の応用
2. **新概念は1節に1つ** — 例:「内積」の節ではベクトルは既習前提にできるよう、必ず前の節で済ませておく
3. **数式より先に体験** — どの節も「デモを触る→現象を言葉にする→最後に式で書くとこうなる」の順。式が主役ではなく、式は「体験のメモ書き」という位置づけで提示する
4. **前提リンクの明示** — 各節の冒頭に「この節で使う数学:内積(第3章4節)」と表示し、クリックで該当節の要点カード(ポップアップ)を再表示できる
5. **つまずき回避コラム** — 挫折しやすいポイント(関数・三角関数・内積・pow・マイクロファセット・SH)には「つまずいたらここに戻る」コラムを必ず置く
6. **難しい話は逃がす** — 厳密な話・発展的な話は折りたたみ(「もっと知りたい人へ」)に隔離し、本文は最短の理解ルートだけにする
7. **短い節を多く** — 1節は5〜10分で読み終わる分量。達成感を細かく刻む

## 4. 学習カリキュラム(コンテンツ構成)

各章は「導入(なぜ必要か)→ 直感デモ → 数式の読み方 → パラメータ実験 → まとめ・確認クイズ」の共通フォーマットで構成する。

**第1章〜第3章が数学の準備、第4章からがCGの内容。** 数学パートに全体の約4割の分量を割く。

CGパートはさらに2段構えになっている:**第4〜8章が「直接光」**(ライトから直接届く光の反射)、**第9〜10章が「間接光」**(周囲で跳ね返って回り込む光)。直接光を完全に理解してから間接光に入る順序にすることで、「直接光だけだと足りない → だから間接光を足す」という動機づけを自然に作る。

### 第0章 イントロダクション
- ライティングとは何か:「光がどう跳ね返って目に届くか」の計算だと説明
- 「最終的にこれが理解できるようになる」ゴールのデモ(PBRの球)を最初に見せて、各パラメータがどの章に対応するかを地図として示す
- インタラクティブUIの使い方チュートリアル

### 第1章 数のきほん復習 ★☆☆(中学数学の思い出し)
CGで使う範囲だけを、デモ付きで思い出す章。「わかる人は飛ばしてOK」と明示する。

1. **0〜1の世界** — CGでは明るさも色も「0=なし、1=最大」で表す。割合・百分率の復習(0.5=50%)。スライダーで球の明るさを0〜1で操作する最初のデモ
2. **マイナスの数** — 「反対向き」を表す道具として復習(のちのベクトル・内積の布石)
3. **座標** — (x, y) の読み方。方眼紙デモで点を動かす
4. **グラフの読み方** — 「横軸を入れると縦軸が出てくる」。折れ線グラフをなぞって値を読むデモ
5. **関数=入力→出力の箱** — 「値を入れると値が返ってくる機械」として説明。y = 2x をスライダーで体感。※「関数」という言葉への抵抗を最初に解いておく(以降すべての土台)
6. **直角三角形と斜めの長さ** — 三平方の定理の復習。「斜めの長さが計算できる」ことだけ確認(ベクトルの長さの布石)

### 第2章 カーブと角度 ★★☆(高校数学をゼロから・前編)
1. **かけ算とカーブ** — y = x² のグラフ。「同じものを掛けると急カーブになる」を体感
2. **べき乗(pow)** — y = xⁿ の n をスライダーで動かし、カーブが「鋭くなる」感覚を体感(スペキュラーの布石)。指数の記法はここで初めて導入
3. **角度の測り方** — 度(°)の復習と、ラジアンの紹介(「円の半径何個ぶん回ったか」)。※以降の本文は度数で通し、ラジアンはコード対応表でのみ使う
4. **sin と cos** — 「円の上を回る点の、縦の位置と横の位置」として導入。角度スライダーで円上の点とグラフが連動するデモ。三角形からではなく円から入る(暗記not必要、読めればOKという到達目標を明示)
5. **cosと「傾きに当たる光」** — 面を傾けると受ける光が減ることを、板に当たる平行線の本数デモで体感(Lambertの直感をここで先に作っておく)

### 第3章 ベクトル ★★☆(高校数学をゼロから・後編)
1. **ベクトル=向きと長さを持つ矢印** — 2Dでドラッグして作るデモ。成分表示 (x, y) との対応
2. **ベクトルの長さ** — 第1章6節の三平方の定理をそのまま使う(既習の再利用を明示)
3. **正規化** — 長さを1にする意味。「向きだけ知りたいときの下ごしらえ」
4. **内積(dot)** — 本サイト最重要。「2つの矢印がどれだけ同じ方向を向いているか(-1〜1)」を2Dデモで体感。第2章4節のcosθとの関係もここで
5. **クランプ / saturate** — 「マイナスは0に切り捨てる」がなぜ必要か
6. **線形補間(lerp)** — 2つの値を混ぜる。金属・非金属の混合の布石
7. **3Dへの拡張** — 「成分が1つ増えるだけで、考え方は同じ」と確認するだけの短い節

### 第4章 ライティングの登場人物 ★★★
- 法線(N)、ライト方向(L)、視線方向(V)、ハーフベクトル(H)、反射ベクトル(R)
- 3Dシーンで各ベクトルを矢印表示し、ライトやカメラをドラッグすると矢印が追従するデモ
- ライトの種類:ディレクショナル / ポイント / スポット(減衰の式も体感)

### 第5章 拡散反射(Diffuse) ★★★
1. **Lambert** — `diffuse = max(0, dot(N, L))`
   - 「面が光に正対するほど明るい」を、第2章5節の板デモ・内積デモ・3D球の3点セットで見せる
2. **Half Lambert** — 陰の落ち方を柔らかくする補正。ゲームらしい「絵作りのための嘘」の例として紹介
3. アルベド(基本色)との掛け算、ライトカラー・強度

### 第6章 鏡面反射(Specular)— 古典編 ★★★
1. **Phong** — `pow(max(0, dot(R, V)), shininess)`
   - shininessスライダーでハイライトが「絞られる」のをグラフと3Dで体感(第2章2節のpowの再登場)
2. **Blinn-Phong** — ハーフベクトルHの導入。「なぜHを使うと計算が楽で見た目も良いか」
3. PhongとBlinn-Phongの見た目比較(切り替えトグル)

### 第7章 PBRの考え方 ★★★
1. なぜPBR(物理ベースレンダリング)が主流になったか(直感的な歴史の話)
2. **エネルギー保存則** — 「入ってきた光より多くは返せない」
3. **マイクロファセット理論** — 表面は微小な鏡の集まり。ラフネスの正体
4. **フレネル** — 「浅い角度ほど鏡になる」を水面の写真とデモで体感。Schlick近似
5. **メタリック/ラフネス ワークフロー** — 2つのスライダーで材質が作れることを体感

### 第8章 BRDF(Cook-Torrance) ★★★
`f = (D · F · G) / (4 · dot(N,L) · dot(N,V))` を分解して1項ずつ学ぶ:

1. **D(法線分布関数 / GGX)** — ラフネスとハイライト形状の関係。GGXカーブをグラフで操作
2. **F(フレネル / Schlick)** — F0の意味、金属と非金属のF0の違い
3. **G(幾何減衰 / Smith)** — マイクロファセット同士の遮蔽。ざらざら面が暗くなる理由
4. **分母の意味** — 正規化(明るさの帳尻合わせ)であることを平易に説明。※式の導出は追わず「役割」の理解に留めると明示
5. **全部合体** — D/F/Gを個別ON/OFFできる3Dデモで、各項の寄与を見る

### 第9章 間接光(インダイレクトライト)入門 ★★★
「直接光だけでは足りない」を体感し、一番安い間接光=スカイライトまでを扱う章。

1. **なぜ間接光が要るか** — 直接光だけの球は「影が真っ黒でのっぺり」する。現実は光が壁や床で跳ね返って影の中にも回り込む。**直接光のみ / 間接光あり を切り替える比較デモ**で違いを体感(この章の動機づけ)
2. **アンビエント(環境光)** — 一番ざっくりした間接光。「全体に一定の明るさを足すだけ」。安いが平面的、という限界も同時に見せる
3. **スカイライト(半球ライト)** — 空からの光(上)と地面からの照り返し(下)で色を変える。式:`ambient = lerp(地面色, 空色, N.y * 0.5 + 0.5)`。第3章6節の**lerp**と第4章の**法線N**の再利用であることを明示。空の色・地面の色スライダーで、球の上下で色が変わるのを3Dで体感
4. **アンビエント vs スカイライト** — 「定数」と「向きで変わる」の違いを比較トグルで
5. **なぜレイトレを使わないか** — リアルタイムでは全方向に光線を飛ばすのは高コスト。**事前に計算した近似で十分それっぽくなる**という発想を紹介(次章SHへの橋渡し。※本サイトがレイトレを扱わない理由もここで説明)

### 第10章 ライトプローブと球面調和関数(SH) ★★★
動くキャラクターにも周囲の間接光を安く適用する、ゲームの定番技術。SHは数学的には重いので、**「係数の意味と役割」だけを直感で理解する**方針(第8章の分母と同じ扱い。厳密な定義・積分は折りたたみに隔離)。

1. **ライトプローブとは** — 「その場所に立ったら周囲からどんな光が来るか」を事前に記録した点。部屋にたくさん置いておき、動くキャラの位置に応じて周囲光を補間する、というイメージをデモで示す
2. **周囲の光を「要約」したい** — 全方向から来る光は、いわば360度のパノラマ写真。そのまま持つと重い。**少ない数字のセットに圧縮したい**という問題設定を、パノラマ画像を粗くしていくデモで体感
3. **SH(球面調和関数)の直感** — 「方向ごとの明るさ」を少数の係数で表す“要約の道具”。ツマミを足すほど細かく表現できる:
   - **L0(1個)** = 全方向の平均の明るさ = 実はアンビエントと同じ
   - **L1(+3個)** = 「どっち側が明るいか」のゆるい傾き(上が明るい/右が明るい 等)
   - リアルタイムではL1まで(色ごとに計4個)で十分、と割り切る
4. **SH係数をいじるデモ** — 係数スライダーを動かすと、球への周囲光の当たり方(ゆるい陰り)が変わる。**「パノラマ環境 → SH係数に要約 → 球へ適用」の3段**を並べて可視化し、要約でどこまで再現できる/できないかを体感
5. **数学の逃がし方とまとめ** — 基底関数の定義・球面上の積分は「もっと知りたい人へ」に隔離。**つまずき回避コラム**「SHがわからなくなったら:要は“周囲の明るさを数個のツマミで表す圧縮”」を設置

## 5. インタラクティブ要素の仕様

### 5.1 共通UIコンポーネント

| コンポーネント | 説明 |
|---|---|
| **パラメータスライダー** | 値・範囲・現在値を表示。ドラッグ中はデモがリアルタイム更新。ダブルクリックで初期値に戻る |
| **2Dグラフビュー** | 関数カーブ(pow、GGX、フレネル等)を描画。スライダーと連動してカーブが変形。ホバーで座標値表示 |
| **3Dプレビュー** | 球・平面等のプリミティブに、その章までに学んだライティングを適用。ライト/カメラをドラッグで回せる |
| **ベクトル操作デモ(2D)** | 矢印をドラッグして内積・正規化などを体感。数値がリアルタイム表示 |
| **数式ビュー** | KaTeX等で数式を整形表示。**数式の各項にホバーすると、その項の説明と、デモ上の対応部分がハイライトされる** |
| **数値の連動表示** | スライダー操作時、数式中の該当変数の現在値が数式内にライブ表示される(例: `dot(N, L) = 0.72`) |
| **比較トグル** | Phong vs Blinn-Phong など、2つのモデルを瞬時に切り替え/左右分割表示 |
| **確認クイズ** | 各章末に2〜4問。「スライダーを動かして○○の状態を作ってみよう」という操作型の問題を含む |
| **要点カード** | 各節の要点(1枚絵+3行以内の説明)。前提リンクから呼び出されるポップアップとしても使う |
| **難易度バッジ** | ★☆☆/★★☆/★★★ を章・節見出しに表示 |

### 5.2 3Dプレビューの仕様

- ライティング計算は**サイト内で教える式をそのままシェーダーに書く**(ライブラリ組み込みのライティングは使わない)。学習内容と表示が完全に一致することを保証するため
- 表示切替:最終結果 / 直接光のみ / 間接光のみ / Diffuseのみ / Specularのみ / 各BRDF項のみ(第8章)/ スカイライトのみ・SHのみ(第9〜10章)
- ライト操作:方位・仰角スライダー+ドラッグ操作の両対応
- マテリアル操作:アルベド色、ラフネス、メタリック、F0 等(章の進度に応じて出す)
- 間接光操作(第9〜10章):空の色・地面の色(スカイライト)、SH係数(L0/L1)、ライトプローブ配置のプレビュー

### 5.3 「文系フレンドリー」のためのルール

- 数式を出す前に、必ず日本語の一文で「この式は要するに何をしているか」を書く
- 記号の初出時は必ず読み方を書く(例:「θ(シータ)= 角度を表すときによく使う文字」)
- 中学数学の範囲を超える概念は、必ず第1〜3章のどこかで導入してから使う(本文中で未導入の数学を使ったら仕様バグとみなす)
- 専門用語には初出時にツールチップ(タップ/ホバーで用語集の定義を表示)
- 1画面1概念。スクロールに合わせて段階的にデモが変化する構成も可
- 各章の冒頭に「この章で使う数学:内積(第3章4節)」のように前提リンクを明示
- 厳密さより直感を優先し、厳密な話は折りたたみに隔離する

| プレイグラウンド | 全パラメータを自由に触れる統合デモ(学習後の遊び場) |
| 用語集 / チートシート | 付録 |

### 6.2 章ページの標準レイアウト(PC)

```
+----------------------------------------------------------+
| ヘッダー(章タイトル / 章ナビゲーション / 進捗)              |
+---------------------------+------------------------------+
|                           |                              |
|  本文(説明・数式)         |  デモパネル(sticky)          |
|  スクロールする            |  - 3Dプレビュー or 2Dグラフ    |
|                           |  - スライダー群                |
|                           |  - 数値表示                   |
+---------------------------+------------------------------+
```

- 本文のスクロール位置に応じて、右のデモパネルの内容・パラメータ構成が切り替わる(scrollytelling方式)
- モバイルでは、デモを本文中にインライン埋め込み(sticky無し)に切り替え

## 7. 技術仕様

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | **Vite + TypeScript(SvelteかReactかは実装開始時に決定)** | 静的サイトとして軽量にビルドできる。型があると数式実装のミスを防げる |
| 3D描画 | **Three.js + カスタムシェーダー(GLSL)** | 教える式をそのままGLSLで書けて学習内容と一致させられる。WebGL2ベース |
| 2Dグラフ | **Canvas 2D または SVG の自作軽量コンポーネント** | 60fpsでカーブを変形させたいので、重いグラフライブラリは避ける |
| 数式表示 | **KaTeX** | 高速・軽量。項単位のホバー連動はカスタム実装 |
| 状態管理 | フレームワーク標準(store等)。スライダー値→数式表示→シェーダーuniformを単一ソースで同期 | |
| 進捗保存 | localStorage(アカウント不要) | |
| ホスティング | GitHub Pages(静的サイト) | このリポジトリから直接デプロイできる |
| 対応ブラウザ | 最新のChrome / Edge / Firefox / Safari。WebGL2必須(非対応時は静的画像+メッセージ表示) | |

### 7.1 非機能要件

- デモ操作は60fpsを目標(最低30fps)。スライダー操作のレスポンスを最優先
- 初回ロード:章ページ単位でコード分割し、3Dアセットは遅延読み込み
- アクセシビリティ:スライダーはキーボード操作可、色だけに依存しない表示、代替テキスト
- 言語:日本語のみ(将来の英語化を見据え、文言は定数ファイルに分離)

## 9. スコープ外(今回はやらないこと)

- ユーザー登録・サーバーサイド機能
- **レイトレーシング / パストレーシング / 厳密なGI(大域照明)の計算**(意図的に扱わない)。間接光はリアルタイム向けの近似=スカイライトと事前計算SHライトプローブ(拡散側)に絞る。理由も第9章5節で学習者に説明する
- スペキュラIBL(鏡面側の間接反射)、シャドウマッピングの詳細(第11章で概念紹介にとどめる)
- SHの厳密な数学(基底関数の定義・球面積分・回転)。第10章では係数の意味・役割の直感までとし、詳細は折りたたみに隔離
- シェーダー言語そのものの文法講座(コード対応表で触れるのみ)
- 多言語対応(構造だけ準備)
